MDX is useful when text alone is not enough. I still default to plain Markdown, but I switch to MDX for posts that benefit from reusable presentation elements.

## My rule of thumb

- Use Markdown for ideas and narrative.
- Use MDX for repeated UI patterns like callouts, comparison blocks, and reference cards.

This keeps content maintainable while still allowing richer technical posts.

## Example: inline navigation component

In this project I can import existing Astro components directly inside a post:

import HeaderLink from '../../components/HeaderLink.astro';

<HeaderLink href="/blog">Browse all blog posts</HeaderLink>

## Final advice

If you are starting from scratch, write your first five posts in Markdown. Add MDX only when repeated patterns clearly appear.
